# 面向低空智联网联邦学习任务的 A3C 无人机路由优化方案

## 1. 课题定位

本课题的核心不是单纯做“最短路径路由”，而是在低空智联网中，为联邦学习任务设计一种能够适应动态网络状态的智能路由策略。开题报告中的目标可以概括为：

- 面向低空智联网中的无人机、地面基站、中继路由器和边缘聚合服务器，建立动态通信网络模型。
- 面向联邦学习的模型上传、模型聚合、全局模型分发流程，降低单轮训练聚合总时延。
- 使用强化学习，重点是 A3C 的异步并行训练机制，让路由策略能够感知链路带宽、节点负载、链路负载和任务数据量。
- 后续再对接 AirSim 与 Mininet，形成“物理运动 - 网络通信 - 算法决策”的联合仿真闭环。

当前代码已经完成了 A3C 路由优化的雏形，尤其是 `uav_routing_multi_gbs` 版本已经实现了多 GBS 依次向聚合服务器上传的环境、奖励函数和 A3C 训练流程。后续工作应在此基础上，把“负载感知路由”升级为“面向 FL 单轮时延的任务感知路由”。

## 3. 建议采用的总体建模

### 3.1 网络模型

将低空智联网抽象为时变有向图：

```text
G_t = (V, E_t)
```

其中：

- `V`：节点集合，包括 UAV、GBS、Router、Edge Server。
- `E_t`：第 `t` 个时隙内可用链路集合。
- `b_l(t)`：链路 `l` 在时隙 `t` 的可用带宽。
- `load_v(t)`：节点 `v` 的当前负载。
- `load_l(t)`：链路 `l` 的当前负载。

结合开题报告和思路梳理文件，建议采用“时隙化准静态”建模：

- 连续真实时间被划分为多个时隙。
- 单个时隙内，拓扑和链路状态近似保持不变。
- 时隙切换时，链路带宽、节点负载、UAV 接入 GBS 关系等网络状态发生变化。
- 强化学习智能体在每个时隙内根据当前网络状态选择路由。

### 3.2 联邦学习任务模型

一轮 FL 任务可拆成三个阶段：

```text
本地训练 -> 模型上传 -> 服务器聚合 -> 全局模型分发
```

路由算法主要优化通信阶段：

1. 上传阶段：多个 UAV/GBS 将本地模型参数上传到边缘服务器。
2. 聚合阶段：服务器完成模型聚合，可先简化为固定计算时间。
3. 分发阶段：服务器将全局模型下发给多个 UAV/GBS。

单轮 FL 时延建议定义为：

```text
T_round = T_up + T_agg + T_down
```

其中：

```text
T_up = max_i T_path(upload_i)
T_down = average_i T_path(download_i)
```

这样做符合联邦学习同步聚合的“掉队者”特点：上传阶段通常由最慢客户端决定全局等待时间；分发阶段可以先用平均时延表示整体下发效率。

路径时延可先用如下简化形式：

```text
T_path(p, M, t) = sum_{l in p} M / R_l(t) + hop_delay * len(p) + queue_delay(l, t)
```

其中：

- `M`：模型参数或数据包大小。
- `R_l(t)`：链路在时隙 `t` 的有效可用带宽。
- `hop_delay`：每跳固定处理时延。
- `queue_delay(l, t)`：由链路负载引入的排队时延，可先用负载比例近似。

## 4. MDP 设计

### 4.1 状态空间

建议将每一步路由决策的状态定义为：

```text
s = (v_c, v_d, M, phase, flow_id, F_t)
```

其中：

- `v_c`：当前节点。
- `v_d`：目标节点。
- `M`：待传输模型参数大小。
- `phase`：当前处于上传阶段还是分发阶段。
- `flow_id`：当前正在处理的 UAV/GBS 编号。
- `F_t`：当前时隙的网络特征。

`F_t` 可以包括：

- 节点类型：UAV、GBS、Router、Server。
- 节点负载：`load_v / cap_v`。
- 链路负载：`load_l / cap_l`。
- 链路剩余带宽：`1 - load_l / cap_l` 或归一化 `R_l / R_max`。
- 邻接关系或可行动作 mask。
- 当前时隙编号或剩余时隙信息。

当前 `uav_routing_multi_gbs` 的状态是：

```text
[current_gbs, current_node, server_id, node_loads, link_loads]
```

后续应扩展为：

```text
[flow_id, current_node, destination_node, model_size, phase, node_features, link_features, time_slot]
```

### 4.2 动作空间

动作定义为选择下一跳节点：

```text
a = next_node
```

动作空间表面上可以仍是全部节点编号，例如 `Discrete(num_nodes)`，但执行时必须使用动作 mask：

- 只能选择当前节点的后继节点。
- 不能选择已经导致明显环路的节点，或者对重复访问节点施加惩罚。
- 如果链路不可用或带宽为 0，应从有效动作中剔除。

后续代码实现中，Actor 输出仍可为全部节点的 logits，但采样和计算 log probability 时都应使用同一套 mask，避免“采样分布”和“训练分布”不一致。

### 4.3 状态转移

执行动作 `a` 后：

- 如果 `a` 是有效邻居，则当前节点更新为 `a`。
- 路径记录追加一跳。
- 累计本条 flow 的路径时延、跳数和即时奖励。
- 如果到达目标节点，该 flow 结束。
- 如果超过最大步数、无有效动作或链路不可用，该 flow 失败。

对于网络负载更新，建议采用两阶段策略：

- 路由搜索过程中先记录路径，不立即永久更新负载。
- flow 成功到达目标后，再将路径上的节点和链路负载提交到共享网络状态。
- flow 失败时不提交负载，但记录失败次数和惩罚。

这样和现有 `uav_routing_shared` 版本的思想一致，也更贴近真实业务：成功传输才占用有效网络资源。

## 5. 奖励函数设计

现有代码已经实现了负载感知局部奖励：

```text
r_route = g_hop + alpha_1 * [g_load(v_j) + g_load(v_i, v_j)] - alpha_2 * c_distr(v_j)
```

建议保留它作为 shaping reward，但加入 FL 时延目标，形成新的复合奖励：

```text
r_step =
    - w_hop * hop_cost
    - w_delay * normalized_link_delay
    + w_load * [g_load(v_j) + g_load(v_i, v_j)]
    - w_balance * c_distr(v_j)
    - w_loop * loop_penalty
```

终止奖励：

```text
r_terminal =
    + R_success,  if reached destination
    - R_fail,     if invalid / timeout / no path
```

FL round 级奖励：

```text
r_round =
    - beta_1 * normalized(T_round)
    + beta_2 * delivery_success_rate
    - beta_3 * load_imbalance
```

实践中可以先分两阶段实现：

1. 第一阶段：只做上传路由，使用 `r_step + r_terminal`。
2. 第二阶段：一个 episode 覆盖完整 FL round，在所有上传路径完成后计算 `T_up`，再加入 round 级奖励。
3. 第三阶段：加入分发路径，计算完整 `T_round = T_up + T_agg + T_down`。

## 6. Episode 设计

建议将一个 episode 定义为一个时隙内的一轮 FL 通信任务。

简化版：

```text
1 episode = 3 个 GBS 依次上传到 Server
```

这正是当前 `uav_routing_multi_gbs` 的设计，适合作为近期代码起点。

论文增强版：

```text
1 episode = 1 个时隙内的一轮 FL 通信
├── 多个 UAV/GBS 上传本地模型
├── 服务器聚合
└── 服务器向多个 UAV/GBS 分发全局模型
```

训练时，每个 Worker 可以在不同随机网络状态、不同流量负载、不同模型大小和不同 UAV 接入关系下并行探索，最后异步更新全局 Actor-Critic 网络。

## 7. A3C 算法设计

### 7.1 网络结构

建议先沿用当前代码的简单全连接结构：

```text
Actor:
state_dim -> hidden_dim -> action_dim

Critic:
state_dim -> hidden_dim -> 1
```

后续如果论文需要更强的创新性，可以扩展为：

- 用图神经网络编码拓扑。
- 用注意力机制关注当前节点邻居。
- 用共享编码器 + Actor/Critic 双头结构减少参数量。

但现阶段不建议一开始就上复杂网络。先保证 MDP、奖励函数、时延模型和实验指标完整。

### 7.2 训练流程

A3C 训练流程建议如下：

```text
初始化全局 Actor 和 Critic
初始化共享优化器
启动多个 Worker

每个 Worker 循环：
    重置环境，生成一个 FL round 或多 GBS 上传任务
    从全局网络同步本地网络参数
    while episode 未结束：
        获取状态 s
        根据动作 mask 从 Actor 策略采样下一跳 a
        环境执行 step(a)
        保存 s, a, r, s'
    计算 n-step return / advantage
    更新全局 Actor 和 Critic
    记录 episode 奖励、成功率、时延、路径
```

### 7.3 必须修正的训练细节

当前代码后续建议修正：

- Worker 内不要直接复用主进程传入的同一个 env，应在 Worker 内创建独立 env。
- 网络负载等全局状态使用 `SharedNetworkState` 或明确的 episode 内本地状态，不能混在一起。
- 动作 mask 要同时用于采样和 log probability 计算。
- 折扣回报计算应避免重复把即时奖励加两次。
- 日志、模型、曲线输出统一放到 `outputs/` 目录。
- 参数集中到配置文件，避免代码与文档不一致。

## 9. 实验设计

### 9.1 对比算法

建议至少设置以下基线：

- Random：随机选择有效下一跳。
- Shortest Path：最短跳数路径。
- Load-aware Dijkstra：以链路时延或负载为边权的 Dijkstra。
- A3C-routing：当前强化学习方案。

### 9.2 指标

论文实验建议记录：

- 平均 episode reward。
- 路由成功率。
- 平均路径跳数。
- 上传阶段最大时延 `T_up`。
- 分发阶段平均时延 `T_down`。
- FL 单轮总时延 `T_round`。
- 链路平均负载和负载方差。
- 节点负载均衡程度。
- 算法收敛曲线。

### 9.3 场景变量

可以逐步增加难度：

- UAV/GBS 数量变化。
- 模型参数大小变化。
- 链路带宽范围变化。
- 背景负载变化。
- 拓扑动态变化频率变化。
- 最大步数限制变化。

## 10. 阶段性实现路线

### 阶段一：整理并跑通当前主线

目标：得到一个稳定、可复现的 A3C 多 GBS 上传版本。

任务：

- 以 `code/uav_routing_multi_gbs` 为基础。
- 删除或忽略无关旧版本干扰。
- 统一节点容量、奖励参数、最大步数等配置。
- 跑通环境测试和短轮训练。
- 输出奖励曲线、路径日志、成功率统计。

### 阶段二：加入共享网络状态

目标：让多个 Worker 能看到一致的全局网络负载。

任务：

- 参考原目录 `uav_routing_shared/SharedNetworkState.py`。
- 每个 Worker 创建自己的环境实例。
- 环境内部读取共享节点负载和链路负载。
- 成功传输后再提交负载更新。

### 阶段三：加入 FL 时延模型

目标：让 reward 和论文目标真正对齐。

任务：

- 增加 `model_size`。
- 增加链路有效带宽计算。
- 增加单条路径时延计算。
- 上传阶段使用所有客户端路径时延的最大值。
- 分发阶段使用所有客户端路径时延的平均值。
- 将 `T_round` 作为 episode 级指标和奖励的一部分。

### 阶段四：加入完整上传 + 分发流程

目标：实现完整 FL round。

任务：

- 上传：UAV/GBS -> Server。
- 聚合：固定聚合时延或按模型数量计算。
- 分发：Server -> UAV/GBS。
- 分别统计 `T_up`、`T_agg`、`T_down`、`T_round`。

### 阶段五：实验与论文图表

目标：形成论文可用实验结果。

任务：

- 加入基线算法。
- 多随机种子实验。
- 绘制奖励曲线、成功率曲线、时延对比图、负载均衡图。
- 总结 A3C 相比传统方法在动态负载下的优势。

## 11. 当前最合理的论文叙述线

建议论文中这样讲：

1. 低空智联网中的 FL 任务受限于通信时延，尤其是同步聚合中的掉队者问题。
2. 传统最短路径算法只关注跳数，难以适应链路带宽和节点负载动态变化。
3. 将 FL 模型参数传输抽象为时隙化动态路由问题，并建模为 MDP。
4. 设计状态空间、动作空间和耦合时延/负载的复合奖励函数。
5. 使用 A3C 的多 Worker 异步探索提高训练效率和环境适应性。
6. 实验中与最短路径、随机策略、负载感知 Dijkstra 对比，证明 A3C 能降低 FL 单轮时延并改善负载均衡。
