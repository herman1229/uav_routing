# A3C多GBS无人机路由优化项目文档

## 项目概述

本项目实现了基于A3C算法的多无人机并发路由优化系统。**核心特点是在一个Episode内依次完成3个GBS的路由决策**，更真实地模拟多无人机协同场景，并且通过共享环境状态实现真实的负载感知。

## 核心设计理念

### Episode = 3个GBS的完整决策过程

```
1个Episode包含:
    ├─ GBS0的路由决策（最多100步）
    ├─ GBS1的路由决策（最多100步）
    └─ GBS2的路由决策（最多100步）

总奖励 = GBS0奖励 + GBS1奖励 + GBS2奖励
```

**设计优势**：
- ✅ 模拟真实的多GBS协同场景
- ✅ 3个GBS在同一网络状态下依次决策
- ✅ 后续GBS能看到前面GBS对网络的影响
- ✅ 更符合实际的联邦学习参数聚合场景

## 系统架构

### 1. 网络拓扑（有向图）

```
节点组成：
  - GBS: 节点0-2（起点）
  - Router: 节点3-6（中继）
  - Server: 节点7（终点）

连接关系（单向）：
  GBS0 (0) ──→ Router1 (3), Router2 (4)
  GBS1 (1) ──→ Router2 (4), Router3 (5)
  GBS2 (2) ──→ Router3 (5), Router4 (6)
  
  Router间双向：
  Router1 (3) ←→ Router2 (4) ←→ Router3 (5) ←→ Router4 (6)
  
  Router → Server（单向）：
  Router3 (5) ──→ Server (7)
  Router4 (6) ──→ Server (7)
```

**路径示例**：
- GBS0 → Router1 → Router3 → Server (3跳)
- GBS1 → Router3 → Server (2跳)
- GBS2 → Router3 → Server (2跳)

## 核心组件

### 1. 环境类（A3C_UAV_Routing_Env.py）

#### 状态空间（24维）

```python
观察向量 = [
    current_gbs_id,      # 当前决策的GBS编号(0-2)  [1维]
    current_node,        # 当前所在节点(0-7)        [1维]
    target_node,         # 目标节点(7)             [1维]
    network_features     # 网络特征向量            [21维]
]

network_features = [
    节点负载(8维),   # 每个节点的流表项使用率
    链路负载(13维)   # 每条边的带宽使用率
]
```

#### 动作空间

```python
动作 = 选择下一个转发节点(0-7)
约束 = 只能选择当前节点的后继节点（有向图）
```

#### Episode流程

```python
Episode开始: reset()
    ↓
GBS0决策阶段:
    current_gbs = 0
    current_node = 0
    ├─ Step 1: 选择动作 → 执行 → 获得奖励
    ├─ Step 2: 选择动作 → 执行 → 获得奖励
    └─ ... 直到到达Server或达到100步
    ↓
    ├─ 成功: 更新网络负载, 累积奖励
    └─ 失败: 不更新网络, 奖励=-10
    ↓
GBS1决策阶段:
    current_gbs = 1
    current_node = 1
    （能看到GBS0对网络的影响）
    ├─ Step 1: ...
    └─ ... 直到完成
    ↓
GBS2决策阶段:
    current_gbs = 2
    current_node = 2
    （能看到GBS0和GBS1对网络的影响）
    ├─ Step 1: ...
    └─ ... 直到完成
    ↓
Episode结束:
    总奖励 = GBS0奖励 + GBS1奖励 + GBS2奖励
    执行反向传播和参数更新
```

#### 奖励机制

**单个GBS的奖励**：
```python
if 成功到达Server:
    GBS奖励 = Σ(每步奖励)
    # 每步奖励 = -1 + 0.4*(节点负载奖励 + 链路负载奖励) - 0.1*分布成本
    # 到达Server的步: +10
    # 典型值: 5 到 10

if 达到100步未到达:
    GBS奖励 = -10（失败惩罚）
```

**Episode总奖励**：
```python
Episode奖励 = GBS0奖励 + GBS1奖励 + GBS2奖励

最好情况: 3个GBS都成功，2-3跳
    Episode奖励 ≈ 8*3 = 24

一般情况: 3个GBS都成功，3-5跳
    Episode奖励 ≈ 6*3 = 18

部分失败: 2个成功，1个失败
    Episode奖励 ≈ 7*2 + (-10) = 4

全部失败: 3个GBS都超时
    Episode奖励 = -10*3 = -30
```

### 2. A3C训练框架（A3C_UAV_Routing.py）

#### Worker工作流程

```python
while global_episode < 1200:
    # 1. 重置环境
    state = env.reset()
    
    # 2. 完成3个GBS的决策（在一个episode内）
    while not all_gbs_done:
        action = take_action(state)
        next_state, reward, terminated, truncated, info = env.step(action)
        
        # 收集经验
        states.append(state)
        actions.append(action)
        rewards.append(reward)
        
        state = next_state
        
        if info['all_gbs_done']:
            break
    
    # 3. 获取episode总奖励
    episode_reward = env.get_episode_reward()
    
    # 4. 反向传播（基于整个episode的所有step）
    update(transition_dict)
    
    # 5. 同步参数
    sync_with_global()
```

#### 关键改进

**旧版本**：
```
1个Episode = 1个GBS的决策
奖励 = 单个GBS的奖励
```

**新版本**：
```
1个Episode = 3个GBS的依次决策
奖励 = 3个GBS奖励之和
```

## 网络负载更新机制

### 负载累积过程

```python
Episode开始:
  初始负载: node_load = {0: 15, 1: 12, ..., 5: 18, ...}

GBS0决策: 0 → 3 → 5 → 7 (成功)
  ↓ 更新负载
  node_load = {0: 16, 1: 12, ..., 3: 17, 5: 19, 7: 16}
  link_load[(0,3)] += 0.08
  link_load[(3,5)] += 0.08
  link_load[(5,7)] += 0.08

GBS1决策: 1 → 5 → 7 (成功，但看到Router3负载已经变高)
  可能选择: 1 → 4 → 6 → 7（避开Router3）
  ↓ 更新负载
  node_load[4] += 1
  node_load[6] += 1
  ...

GBS2决策: 2 → ? (看到Router3和Router4的负载情况)
  智能选择低负载路径
```

### 失败不更新

```python
GBS0决策: 100步后仍未到达 (失败)
  ✗ 不更新 node_load
  ✗ 不更新 link_load
  ✓ GBS0奖励 = -10

GBS1和GBS2: 在GBS0失败的情况下继续决策
  网络负载保持原状（不包含GBS0的影响）
```

## 关键方法说明

### env.reset()
```python
def reset(self, seed=None):
    """
    重置环境，开始新的episode
    
    Returns:
        obs: 初始观察（GBS0的起始状态）
    """
```

### env.step(action)
```python
def step(self, action):
    """
    执行一步动作
    
    自动管理:
      - 当前是哪个GBS在决策
      - 该GBS是否完成
      - 是否需要切换到下一个GBS
      - 网络负载的更新
    
    Returns:
        obs: 新观察
        reward: 即时奖励
        terminated: Episode是否结束（3个GBS都完成）
        truncated: 是否超时
        info: 详细信息
    """
```

### env.get_episode_reward()
```python
def get_episode_reward(self):
    """
    获取episode总奖励（3个GBS之和）
    
    Returns:
        float: 总奖励
    """
```

### env.get_episode_info()
```python
def get_episode_info(self):
    """
    获取episode详细信息
```

## 技术实现细节

### GBS切换逻辑

```python
# 在env.step()中
if gbs_terminated or gbs_truncated:
    if gbs_terminated:
        # 成功：更新网络负载
        self._update_network_load(path_nodes)
    else:
        # 失败：设置惩罚奖励
        self.gbs_rewards[self.current_gbs] = -10
    
    # 切换到下一个GBS
    self.current_gbs += 1
    
    if self.current_gbs >= self.num_gbs:
        # 所有GBS完成，Episode结束
        return ..., terminated=True, ...
    else:
        # 开始下一个GBS的决策
        self.current_node = self.current_gbs
        return ..., terminated=False, ...
```

```python
obs = [
```

## 总结

本系统实现了**Episode内多GBS依次决策**的无人机路由优化，核心特点：

1. ✅ **一个Episode = 3个GBS完整决策**
2. ✅ **网络负载依次累积**（后续GBS看到前面GBS的影响）
3. ✅ **成功才更新网络**（失败给予惩罚但不占用资源）
4. ✅ **总奖励 = 3个GBS奖励之和**
5. ✅ **有向拓扑**（GBS→Router单向）

这是最接近实际联邦学习参数聚合场景的设计！
