# A3C多GBS无人机路由优化完整系统说明

## 系统概览

本系统实现了基于A3C算法的多无人机协同路由优化，在一个Episode内依次完成3个GBS的路由决策，真实模拟联邦学习场景下的参数聚合传输。

## 常见问题

### Q1: 为什么观察维度是29而不是23？

**A**: 增加了`current_gbs_id`（当前是哪个GBS在决策）
```
23维: current_node + target + network_features
29维: current_gbs + current_node + target + network_features
```

### Q2: 3个GBS的顺序固定吗？

**A**: 是的，固定顺序GBS0 → GBS1 → GBS2
- 这样可以稳定学习
- 也可以修改为随机顺序

### Q3: 如果GBS1失败，GBS2还会继续吗？

**A**: 会继续！
```
GBS0: 成功 (更新网络)
GBS1: 失败 (不更新，奖励=-10)
GBS2: 继续决策 (在GBS0更新后的网络上)
```

### Q4: Episode奖励可能是负数吗？

**A**: 可以！
```
如果3个GBS都失败:
Episode奖励 = -10 + (-10) + (-10) = -30

如果2个失败1个成功:
Episode奖励 = -10 + 8 + (-10) = -12
```

### Q5: 如何调整让训练更容易成功？

**A**: 
```python
# 方法1: 增加步数限制
self.max_steps_per_gbs = 150

# 方法2: 降低初始负载
self.node_load = {i: random.randint(5, 10) for i in range(8)}

# 方法3: 减少失败惩罚
self.failure_penalty = -5
```

   ```
   初期平均: 5-10
   中期平均: 12-18

   ```

3. **路径多样化**
   ```
   初期: 都走Router5（最短路径）
   后期: 分散到Router3-6（负载均衡）
   ```

4. **步数优化**
   ```
   初期: 平均5-8步/GBS
   后期: 平均2-3步/GBS
   ```

### 失败的训练可能显示

1. **奖励不收敛**
   - 原因: 学习率过高/过低
   - 解决: 调整学习率

2. **成功率一直很低**
   - 原因: max_steps太小或初始负载太高
   - 解决: 增加max_steps或降低初始负载

3. **奖励收敛到负数**
   - 原因: 奖励函数设计问题
   - 解决: 调整奖励权重或惩罚值

## 技术亮点

### 1. 真实的并发模拟

```
不是简单的多线程，而是:
  - 在同一个Episode内
  - 在同一个网络状态下
  - 依次决策，负载累积
  
这才是真实的多GBS并发场景！
```

### 2. 智能的负载感知

```
后续GBS能看到前面GBS的影响:
  GBS1观察 → 包含GBS0造成的负载变化
  GBS2观察 → 包含GBS0和GBS1的累积影响
  
学习结果: 自动避开高负载路径
```

### 3. 合理的失败处理

```
失败不更新网络:
  - 符合现实（失败的传输不占用资源）
  - 给予惩罚（-10），鼓励成功
  - 不影响后续GBS的决策
```
