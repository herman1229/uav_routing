# A3C多GBS路由优化设计说明

## 设计思路

基于您10.15版本的代码，进行了重大改进，实现了**Episode内多GBS依次决策**的训练模式。

---

## 核心改进

### 1. Episode结构变化

#### 旧版本（10.15）
```
1个Episode = 1个Worker的1个GBS决策
├─ 起点: 随机GBS(0-2)
├─ 决策: 选择路径到Server
└─ 奖励: 单个路径的奖励
```

#### 新版本（multi_gbs）
```
1个Episode = 3个GBS的完整决策周期
├─ GBS0决策: 0 → ... → 7
├─ GBS1决策: 1 → ... → 7（看到GBS0的影响）
└─ GBS2决策: 2 → ... → 7（看到GBS0和GBS1的影响）

总奖励 = GBS0 + GBS1 + GBS2
```

## 详细设计

### 环境状态管理

```python
class UAVRoutingEnv:
    def __init__(self):
        # Episode级别状态
        self.current_gbs = 0          # 当前决策的GBS(0,1,2)
        self.gbs_paths = {}           # {0: path0, 1: path1, 2: path2}
        self.gbs_rewards = {}         # {0: reward0, 1: reward1, 2: reward2}
        self.gbs_success = {}         # {0: True/False, 1: ..., 2: ...}
        self.gbs_step_count = {}      # {0: steps0, 1: steps1, 2: steps2}
```

### Step执行逻辑

```python
def step(self, action):
    # 1. 执行动作
    step_reward = calculate_reward(...)
    
    # 2. 累积当前GBS的奖励
    self.gbs_rewards[self.current_gbs] += step_reward
    
    # 3. 检查当前GBS是否完成
    if reached_server:
        # 成功：更新网络负载
        self._update_network_load(path)
        self.gbs_success[self.current_gbs] = True
        
        # 切换到下一个GBS
        self.current_gbs += 1
        self.current_node = self.current_gbs
        
    elif steps >= 100:
        # 失败：不更新网络，给予惩罚
        self.gbs_rewards[self.current_gbs] = -10
        self.gbs_success[self.current_gbs] = False
        
        # 切换到下一个GBS
        self.current_gbs += 1
        self.current_node = self.current_gbs
    
    # 4. 检查Episode是否结束
    if self.current_gbs >= 3:
        terminated = True  # 3个GBS都完成了
    else:
        terminated = False
    
    return obs, step_reward, terminated, truncated, info
```

## 关键特性

### 1. 依次决策，负载累积

```python
顺序很重要：
  GBS0先决策 → 更新网络
  GBS1后决策 → 看到GBS0的影响
  GBS2最后决策 → 看到GBS0和GBS1的影响

结果：
  后续GBS能够智能避开前面GBS使用的高负载路径
```

### 2. 成功/失败处理

```python
成功（到达Server）:
  ✅ 更新网络负载（节点+1，链路+0.08）
  ✅ 获得正常奖励（约8-10）
  
失败（100步超时）:
  ❌ 不更新网络负载
  ❌ 奖励强制设为-10
  
目的：只有有效传输才占用网络资源
```

### 3. 有向拓扑

```python
GBS → Router: 单向
  ├─ 0 → [3, 4]
  ├─ 1 → [4, 5]
  └─ 2 → [5, 6]
  
Router ↔ Router: 双向
  ├─ 3 ↔ 4 ↔ 5 ↔ 6
  
Router → Server: 单向
  ├─ 5 → 7
  └─ 6 → 7
```

## 训练机制

### Worker训练循环

```python
while global_episode < 1200:
    # 1. 重置环境（开始新episode）
    state = env.reset()
    current_gbs = 0
    
    # 2. 依次完成3个GBS的决策
    while current_gbs < 3:
        # 为当前GBS选择动作
        action = take_action(state)
        next_state, reward, terminated, truncated, info = env.step(action)
        
        # 收集经验
        states.append(state)
        actions.append(action)
        rewards.append(reward)
        
        state = next_state
        
        # 检查是否所有GBS都完成
        if info['all_gbs_done']:
            break
    
    # 3. 获取episode总奖励
    episode_reward = env.get_episode_reward()  # 3个GBS奖励之和
    
    # 4. 反向传播（使用整个episode的经验）
    update(transition_dict)
    
    # 5. 参数同步
    sync_with_global()
```

### 经验收集

```python
transition_dict = {
    'states': [s1, s2, s3, ..., s_n],     # GBS0的steps + GBS1的steps + GBS2的steps
    'actions': [a1, a2, a3, ..., a_n],
    'rewards': [r1, r2, r3, ..., r_n]
}

n = GBS0_steps + GBS1_steps + GBS2_steps
典型值: n ≈ 3 + 2 + 2 = 7步
```

## 奖励计算详解

### 即时奖励（每步）

```python
reward = g_hop + α₁[g_load(node) + g_load(link)] - α₂*c_distr

参数:
  g_hop = -1
  α₁ = 0.4
  α₂ = 0.1

特殊情况:
  到达Server: +10
  无效动作: -10
```

### GBS奖励（单个GBS的所有步骤）

```python
if 成功:
    GBS奖励 = Σ(即时奖励)
    # 典型值: 2跳约9分，3跳约8分，4跳约7分
    
if 失败:
    GBS奖励 = -10（惩罚，不管走了多少步）
```

### Episode奖励（3个GBS）

```python
Episode奖励 = GBS0奖励 + GBS1奖励 + GBS2奖励

奖励范围:
  最优: 3×9 = 27（3个GBS都2跳成功）
  良好: 3×7 = 21（3个GBS都3-4跳成功）
  一般: 2×8 + 1×(-10) = 6（2成功1失败）
  较差: 3×(-10) = -30（3个都失败）
```

## 代码关键部分解析

### 环境切换GBS的逻辑

```python
# env.step()中
if gbs_terminated or gbs_truncated:
    # 处理当前GBS的结束
    if gbs_terminated:
        self._update_network_load(path)  # 成功：更新
    else:
        self.gbs_rewards[self.current_gbs] = -10  # 失败：惩罚
    
    # 切换GBS
    self.current_gbs += 1
    
    if self.current_gbs >= 3:
        return obs, reward, terminated=True, ...  # Episode结束
    else:
        self.current_node = self.current_gbs  # 下一个GBS的起点
        return obs, reward, terminated=False, ...  # 继续
```

### Worker的episode处理

```python
# Worker.run()中
while not episode_terminated:
    action = self.take_action(state)
    next_state, reward, terminated, truncated, info = self.env.step(action)
    
    # 收集所有step的经验
    transition_dict['states'].append(state)
    transition_dict['actions'].append(action)
    transition_dict['rewards'].append(reward)
    
    # 检查所有GBS是否都完成
    if info['all_gbs_done']:
        episode_terminated = True
    
    state = next_state

# Episode结束后
episode_reward = env.get_episode_reward()  # 总奖励
update(transition_dict)  # 一次性更新（基于所有step）
```

## 观察维度说明

### 为什么是29维？

```python
obs_dim = 3 + network_feature_dim
        = 3 + (8 + 18)
        = 29

详细分解:
  3维基础信息:
    - current_gbs_id: 0/1/2
    - current_node: 0-7
    - target_node: 7
  
  8维节点负载:
    - node_load[0]/50
    - node_load[1]/50
    - ...
    - node_load[7]/50
  
  18维链路负载:
    - 18条有向边的负载
    - 包括(0,3), (0,4), (1,4), (1,5), ...
```

**为什么包含current_gbs_id？**
- 告诉agent现在是哪个GBS在决策
- agent可以学习针对不同GBS的不同策略
- 例如：GBS2可能学习到更保守的策略（因为前面GBS已经占用了网络）
